## Overview

Ce projet a été réalisé en **2ᵉ année de BUT Informatique** dans le cadre d’un enseignement de développement web backend.  
L’objectif était de développer une **application web en PHP** en respectant une **architecture MVC (Model – View – Controller)**.

Ce projet marque une étape importante dans la compréhension des **architectures web structurées**, au-delà du PHP procédural.

## Objectifs du projet

- Comprendre et appliquer le modèle MVC
- Séparer la logique métier, l’affichage et le contrôle
- Manipuler une base de données relationnelle
- Créer une application web maintenable et évolutive
- Mettre en pratique les bonnes pratiques du développement backend

## Fonctionnalités principales

- **Gestion des données** via une base de données MySQL
- **Contrôleurs dédiés** pour chaque fonctionnalité
- **Vues dynamiques** générées côté serveur
- **Routing simple** pour gérer la navigation
- **CRUD** (Create, Read, Update, Delete) sur les entités principales

## Technologies utilisées

- **PHP**
- **Architecture MVC**
- **MySQL**
- **HTML / CSS**
- **PDO** pour l’accès à la base de données

## Difficultés rencontrées et apprentissages

La principale difficulté a été de **changer de manière de penser** par rapport au PHP classique.  
Ce projet m’a permis de :
- comprendre l’intérêt de la séparation des responsabilités
- structurer un projet web à moyen terme
- poser des bases solides pour l’apprentissage de frameworks modernes comme Laravel

## Résultat et bilan

L’application répond aux attentes fonctionnelles et respecte les principes du MVC.  
Ce projet a été déterminant dans ma progression en développement web backend et a facilité la transition vers des frameworks plus avancés par la suite.
